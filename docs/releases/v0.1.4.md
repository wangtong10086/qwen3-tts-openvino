# Qwen3-TTS OpenVINO Runtime v0.1.4

This release is the first public package after consolidating the project around
one production runtime path. It is intended for two audiences:

- End users who want a native Linux or Windows sidecar with HTTP/WebSocket APIs.
- Developers who want to export, validate, benchmark, and extend the OpenVINO
  Qwen3-TTS runtime without carrying legacy paths.

## Assets

GitHub Release:

- Linux x64: `qwen3-tts-ov-server-linux-x64-0.1.4-runtime-minimal.tar.zst`
- Windows x64: `qwen3-tts-ov-server-windows-x64-0.1.4-runtime-minimal.zip`

The GitHub packages contain the runtime application only. They do not contain
PyTorch weights, source model weights, OpenVINO IR, generated audio, or
OpenVINO compile caches.

Compiled OpenVINO IR is published separately on Hugging Face:

- Model repo: <https://huggingface.co/waston10086/qwen3-tts-openvino-voice-design>
- Runtime subdirectory: `openvino_realtime/`
- Included modes:
  - `openvino_realtime/voice_design`
  - `openvino_realtime/custom_voice`
  - `openvino_realtime/base` for VoiceClone

The release server downloads the required IR automatically when the local model
directory is missing.

## User-Facing Changes

### Full Sidecar Workflow

The prebuilt runtime now starts as a local sidecar and exposes:

- Web Demo at `http://127.0.0.1:17860/`
- WebSocket streaming endpoint: `/v1/tts/stream`
- HTTP NDJSON streaming endpoint: `/v1/tts/stream`
- Non-streaming WAV endpoint: `/v1/tts`
- OpenAI-compatible speech endpoint: `/v1/audio/speech`
- Model status and one-click download endpoints: `/v1/models` and
  `/v1/models/download`

Typical startup:

```bash
./qwen3-tts-ov-server --device GPU
```

### Three TTS Modes

The public model repo now includes runtime-minimal IR for all supported modes:

- VoiceDesign: `text + instruct + language`
- CustomVoice: `text + speaker + optional instruct + language`
- VoiceClone: `text + ref_audio/ref_text + language`

VoiceClone defaults to the full reference-audio path. `x_vector_only` is no
longer the Web Demo default, so uploaded reference audio is actually used unless
the user explicitly enables the embedding-only comparison mode.

### Improved Web Demo

The Web Demo was replaced with packaged static resources and now supports:

- Model/mode switching from the main title area.
- Short and long default text presets.
- VoiceClone reference-audio upload.
- Direct model download buttons when a mode is missing locally.
- Concurrent request smoke testing from the same screen.
- Request JSON and curl preview.
- Detailed debug metadata, including runtime profile, long-text mode, KV-cache
  settings, stream status, and model availability.

### Long Text Behavior

Long text generation is no longer implemented by text segmentation as the
primary path. The production runtime uses full-context autoregressive generation
with paged KV cache so tone and speaker style remain continuous across long
requests.

## Runtime And Performance Architecture

The public runtime has been collapsed to one production profile:

- `fastest`
- native OpenVINO codegen backend
- paged KV cache
- U8 KV cache by default
- vLLM-like online batching scheduler
- runtime-minimal IR with only the production graph set

The Hugging Face IR intentionally keeps only the validated production graphs:

- text and codec embedding
- `int8_sym_batch_fused_gqa` paged-KV seed graph
- cached subcode graph used by the scheduler
- streaming decoders `c0_t8`, `c25_t12`, and `c25_t24`
- Base/VoiceClone audio graphs: `speech_encoder`, `speaker_encoder`, and
  `code_frame_embedding`

Legacy full decoders, old no-cache paths, fixed-bucket diagnostics, unroll
experiments, old benchmark scripts, and stale alternate graph variants are not
published in the runtime-minimal model repo.

## Windows GPU+NPU Optional Path

The default runtime remains GPU-only. Windows Intel GPU+NPU users can
experiment with heterogeneous execution:

```powershell
.\qwen3-tts-ov-server.exe --device GPU --npu-offload decoder
```

Supported values include `off`, `auto`, `decoder`, `audio`, `all`, and
`require`. The repository includes Windows probe, smoke, benchmark, and analysis
scripts for this path. GitHub Actions builds the Windows runtime, but real NPU
performance validation is expected to run on local Windows hardware with an
actual OpenVINO NPU device.

## Developer Changes

### Repository Cleanup

The repository was reorganized as a source-first OpenVINO project:

- Legacy ONNX/XPU/old runtime paths were removed from the active tree.
- Old ad hoc benchmark and diagnostic scripts were replaced by focused scripts
  for the current production architecture.
- Web assets were moved into `qwen3_tts_ov/web_static/` and packaged with the

## Upgrade Notes

- If you previously downloaded a large or experimental OpenVINO IR directory,
  prefer deleting it and letting the release server download
  `openvino_realtime/` again.
- If a local IR is missing `int8_sym_batch_fused_gqa` or the listed streaming
  decoders, it is not the current runtime-minimal production IR.
- The source repository still ignores `models/`, `openvino/`, `openvino_full/`,
  `outputs/`, generated audio, native build outputs, and release artifacts.
- For offline deployment, predownload the Hugging Face `openvino_realtime/**`
  directory and start the server with `--model-root <path-to-openvino_realtime>`.
